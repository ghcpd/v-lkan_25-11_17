## 💾 数据格式规范

### 实体 (Entity) JSON 格式

```json
{
  "id": "e_1",
  "text": "John Doe",
  "type": "Person",
  "start": 0,
  "end": 8,
  "confidence": 0.92,
  "context": "完整上下文句子"
}
```

**字段说明:**
- `id`: 唯一标识符，用于关系链接
- `text`: 提取的文本内容
- `type`: 实体类型 (10种)
- `start/end`: 文本在原文档中的位置
- `confidence`: 提取的置信度 (0-1)
- `context`: 包含实体的上下文句子

### 关系 (Relation) JSON 格式

```json
{
  "id": "r_1",
  "type": "works_at",
  "head": "e_1",
  "head_text": "John Doe",
  "head_type": "Person",
  "tail": "e_10",
  "tail_text": "OpenAI",
  "tail_type": "Company",
  "confidence": 0.92,
  "evidence": "证据句子"
}
```

**字段说明:**
- `id`: 唯一标识符
- `type`: 关系类型 (8种)
- `head`: 头实体ID
- `tail`: 尾实体ID
- `head_text/type`: 头实体的文本和类型
- `tail_text/type`: 尾实体的文本和类型
- `confidence`: 关系提取的置信度
- `evidence`: 支持该关系的证据文本
